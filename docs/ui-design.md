# Edge 翻译插件 UI 设计说明

## 1. 文档目标

本文档用于定义 Edge 翻译插件 MVP 的 UI 设计方案，覆盖以下内容：

- 插件各入口页面的界面目标。
- 核心交互流程与状态设计。
- 组件拆分和视觉层级。
- 文案风格、反馈方式、异常状态与可访问性要求。

本文档只负责 UI 设计，不包含任何实现代码。

## 2. UI 设计原则

### 2.1 快速触达

- 用户发起翻译的路径必须短。
- 页面选区翻译应尽量在用户当前阅读位置完成，不打断阅读上下文。
- 截图翻译应在最少步骤内进入框选模式，不要求用户手动上传图片。
- 粘贴翻译应在两步之内完成：打开插件、粘贴并点击翻译。

### 2.2 低打扰

- 插件不自动弹出结果。
- 页面仅在用户存在有效选区且愿意翻译时显示轻量触发入口。
- 截图模式只在用户显式点击“开始截图翻译”后进入。
- 浮层尺寸受控，避免遮挡原文大段内容。

### 2.3 信息优先级清晰

- 第一优先级是译文。
- 第二优先级是原文。
- 第三优先级是操作按钮和辅助信息。
- API Key、供应商地址、调试信息不在日常使用界面突出显示。

### 2.4 状态明确

- 用户必须能明确区分空状态、加载态、成功态、失败态。
- 所有操作都应有即时反馈，例如按钮 loading、复制成功提示、连接验证结果。
- 截图模式必须明确区分“待框选”“框选中”“处理中”“完成/失败”。
- 当配置缺失时，必须明确引导用户前往设置页。

### 2.5 与浏览器环境兼容

- UI 需适配 Edge 扩展 popup 的窄视口。
- 页面浮层需适应复杂网页背景与选区位置。
- 截图遮罩需在大多数网页上保持稳定覆盖。
- 默认不依赖网站自身样式，避免出现字体、按钮、颜色被宿主页面污染。

## 3. 信息架构

MVP 中涉及四个主要 UI 场景：

1. 页面文本选区翻译浮层。
2. 页面截图框选模式与结果浮层。
3. 插件 popup。
4. 插件 options 设置页。

### 3.1 页面文本选区翻译入口

- 主目标：在阅读流中快速看翻译。
- 主要任务：触发翻译、阅读译文、复制结果、关闭浮层。
- 使用频率：最高。
- 设计策略：轻量、局部、非沉浸式。

### 3.2 页面截图翻译入口

- 主目标：解决不可选中、不可复制内容的翻译问题。
- 主要任务：启动截图模式、框选区域、查看翻译结果、复制结果。
- 使用频率：中高。
- 设计策略：明确模式切换、局部上传、结果就地展示。

### 3.3 Popup 入口

- 主目标：处理手动粘贴翻译，并提供截图翻译入口。
- 主要任务：在 popup 头部选择目标语言、输入文本、发起翻译、启动截图模式、查看结果。
- 使用频率：中高。
- 设计策略：结构化、表单式、支持较长文本。

### 3.4 Options 入口

- 主目标：管理文本翻译逻辑、截图翻译逻辑和默认语言。
- 主要任务：配置文本翻译模型、选择截图翻译模式，并按实际 pipeline 配置所需模型。
- 使用频率：低，但对首用体验关键。
- 设计策略：逻辑聚合、按需展开、允许复用文本翻译配置，也允许对截图各阶段单独配置。

## 4. 页面文本选区翻译 UI

### 4.1 交互流程

```text
用户选中文本
  -> 选区附近出现轻量触发按钮
  -> 用户点击“翻译”
  -> 如未配置则提示前往设置
  -> 如已配置则显示加载态浮层
  -> 请求成功后展示双语结果
  -> 用户可复制译文 / 重试 / 关闭
```

### 4.2 触发按钮设计

#### 目标

- 只在用户完成选区后出现。
- 视觉足够明显，但不压过网页内容。
- 能在密集页面中稳定定位。

#### 设计建议

- 外观：圆角小按钮。
- 文案：`翻译`。
- 位置：默认出现在选区右上或右下，根据视口空间自动调整。
- 显示时机：仅当选区字符数大于最小阈值且不是纯空白时出现。
- 消失时机：选区取消、点击空白区、点击关闭、滚动超过阈值后隐藏。

### 4.3 浮层设计

```text
+----------------------------------+
| 翻译结果                    [×]  |
|----------------------------------|
| 原文                             |
| Hello world ...                  |
|----------------------------------|
| 译文                             |
| 你好，世界 ...                   |
|----------------------------------|
| [复制译文] [重试]         已完成 |
+----------------------------------+
```

#### 内容层级

- 顶部：标题 + 关闭。
- 中部：原文块。
- 下部：译文块。
- 底部：操作区与状态提示。

#### 定位原则

- 优先贴近选区显示。
- 若右侧空间不足，则切换到左侧或下方。
- 浮层必须避免超出视口。
- 浮层与选区之间应有 8px 至 12px 间距。

### 4.4 选区配置缺失态

若用户点击翻译但尚未完成模型配置：

- 不直接报网络错误。
- 在原位置显示一个轻量提示卡片。
- 主文案：`请先完成文本翻译配置`
- 次级动作：`打开设置`

## 5. 页面截图翻译 UI

### 5.1 交互流程

```text
用户在 popup 中点击“开始截图翻译”
  -> 当前页面进入截图模式
  -> 页面显示半透明遮罩与操作提示
  -> 用户拖拽框选区域
  -> 如未配置或无权限则给出明确提示
  -> 处理中
  -> 请求成功后在区域附近展示双语结果浮层
  -> 用户可复制译文 / 重试 / 关闭
```

### 5.2 截图模式入口

截图翻译的 MVP 入口固定为 `popup` 中的二级操作，不与文本选区按钮混合。

### 5.3 截图遮罩设计

- 全屏半透明暗层。
- 鼠标变为十字准星。
- 顶部或中央显示一行操作说明。
- 按 `Esc` 退出模式。
- 若配置缺失，遮罩不进入，直接在 popup 中提示去设置。

#### 建议文案

- 默认提示：`拖拽选择要翻译的区域，按 Esc 取消`
- 框选中提示：`松开鼠标开始翻译`
- 处理中提示：`正在识别并翻译区域内容…`

### 5.4 区域选择规则

- 仅允许矩形区域。
- 最小框选尺寸应有限制，避免误触。
- 小于最小尺寸时不提交，提示用户重新选择。
- 框选完成后直接进入处理态，不增加确认弹窗。

### 5.5 截图结果浮层

```text
+----------------------------------+
| 截图翻译结果                [×]  |
|----------------------------------|
| 识别原文                         |
| Sign in to continue              |
|----------------------------------|
| 译文                             |
| 登录以继续                       |
|----------------------------------|
| [复制译文] [重试]         已完成 |
+----------------------------------+
```

### 5.6 截图错误与权限提示

建议把截图相关异常区分清楚：

- 文本配置缺失：`请先在设置中完成文本翻译配置`
- 视觉配置缺失：`请先在设置中完成截图翻译配置`
- 文本权限缺失：`请先授权访问文本翻译服务地址`
- 视觉权限缺失：`请先授权访问截图翻译服务地址`
- 截图失败：`当前页面截图失败，请重试`
- 区域无效：`选择区域过小，请重新框选`
- 图片过大：`截图区域过大，请缩小后重试`
- 视觉模型不可用：`当前模型不支持截图翻译`

## 6. Popup UI 设计

### 6.1 使用目标

popup 是两个能力的轻量入口：

- 文本粘贴翻译。
- 截图翻译模式启动。

它也是用户发现“文本翻译是否就绪”和“截图翻译 pipeline 是否完整”的第一入口。

### 6.2 版面结构

```text
+------------------------------------------+
| Edge 翻译插件 [简体中文 v]         [设置] |
|------------------------------------------|
| [文本翻译] [截图翻译]                    |
|------------------------------------------|
| 文本翻译：已就绪 / 未就绪                |
| 截图翻译：直译可用 / 两阶段未完整         |
|------------------------------------------|
| 文本翻译 Tab                             |
| 输入文本                                 |
| +--------------------------------------+ |
| | 粘贴要翻译的内容…                    | |
| +--------------------------------------+ |
| [翻译]                             0/2000 |
|------------------------------------------|
| 原文                                     |
| ...                                      |
|------------------------------------------|
| 译文                                     |
| ...                                      |
|------------------------------------------|
| [复制译文]                               |
+------------------------------------------+
```

截图翻译 Tab：

```text
+------------------------------------------+
| Edge 翻译插件 [简体中文 v]         [设置] |
|------------------------------------------|
| [文本翻译] [截图翻译]                    |
|------------------------------------------|
| 截图翻译                                 |
| 对当前页面进行区域截图翻译               |
| 当前模式：直接多模态 / 提取后翻译         |
| 仅上传你框选的区域                       |
| 本次语言同时作用于截图与粘贴翻译         |
|                                          |
| [开始截图翻译]                           |
|                                          |
| 未配置时：先去设置完成截图翻译配置       |
+------------------------------------------+
```

### 6.3 Popup 状态设计

#### 初始态

- 默认停留在文本翻译 tab。
- popup 标题栏右侧显示目标语言选择，两个 tab 共用同一个当前会话语言。
- tab 切换使用按钮语义，并提供 `role=tablist` / `role=tab` / `aria-selected`。
- 分别显示文本翻译状态和截图翻译状态。

#### 文本未配置态

- 文本翻译 `翻译` 按钮不可直接执行真实请求。
- 主提示：`请先完成文本翻译配置`
- 提供 `打开设置` 快捷入口。

#### 截图未配置态

- `开始截图翻译` 按钮不可进入真实截图流程。
- 若截图模式为直译，主提示：`请先完成截图直译配置`。
- 若截图模式为两阶段，主提示：`请先完成截图提取与截图翻译配置`。
- 提供 `打开设置` 快捷入口。

#### 文本翻译 Loading / Success / Error

- 与原方案一致，但错误提示改为更偏“配置/权限/网络”语义。

#### 截图启动中

- 点击 `开始截图翻译` 后，按钮短暂显示 `正在启动…`。
- popup 头部当前选择的目标语言会随启动消息一并传入截图模式，并继续用于本次框选提交。
- 成功后 popup 自动关闭。

## 7. Options 设置页 UI

### 7.1 页面目标

设置页从“后端设置”调整为“模型供应商设置”，这是整个插件首次可用性的核心页面。

设置入口应以浏览器独立标签页打开，而不是嵌入在受限宽度的扩展管理容器中。

### 7.2 页面结构

- 设置页采用居中单列内容区，桌面端宽度随窗口自适应扩展，内容区最大宽度控制在约 960px。
- 页面缩窄时，状态行需从“标签 + 内容”两栏自动折叠为单列，不允许出现横向滚动。
- 长 URL、长模型名、长英文提示必须优先换行，不得挤压按钮、输入框或状态卡片。

```text
+--------------------------------------------------+
| 设置                                              |
|--------------------------------------------------|
| 文本翻译配置                                      |
| 供应商类型 [OpenAI-compatible v]                  |
| 服务地址 https://api.openai.com/v1                |
| API Key sk-text-****************                  |
| [显示/隐藏]                                       |
| 文本模型 gpt-4.1-mini                             |
| 文本权限：已授权 / 未授权                         |
| 文本状态：已验证 / 未验证                         |
| [请求文本权限] [验证文本翻译]                     |
|--------------------------------------------------|
| 截图翻译配置                                      |
| 模式 [直接多模态翻译 v / 提取后翻译 v]            |
|--------------------------------------------------|
| 当模式 = 直接多模态翻译                           |
| 配置来源 [复用文本翻译配置 / 单独配置]            |
| 若单独配置：                                      |
|   供应商类型 [OpenRouter-compatible v]            |
|   服务地址 https://openrouter.ai/api/v1           |
|   API Key sk-direct-**************                |
|   视觉模型 google/gemini-2.5-flash                |
|   [请求截图直译权限] [验证截图直译]               |
|--------------------------------------------------|
| 当模式 = 提取后翻译                               |
| 提取阶段：                                        |
|   供应商类型 [OpenRouter-compatible v]            |
|   服务地址 https://openrouter.ai/api/v1           |
|   API Key sk-extract-*************                |
|   视觉模型 google/gemini-2.5-flash                |
|   [请求提取权限] [验证截图提取]                   |
| 翻译阶段：                                        |
|   配置来源 [复用文本翻译配置 / 单独配置]          |
|   若单独配置：                                    |
|     供应商类型 [DeepSeek-compatible v]            |
|     服务地址 https://api.deepseek.com/v1          |
|     API Key sk-translate-***********              |
|     文本模型 deepseek-chat                        |
|     [请求截图翻译权限] [验证截图翻译]             |
|--------------------------------------------------|
| 默认目标语言                                      |
| 简体中文                                          |
|--------------------------------------------------|
| 本地存储说明：仅保存在当前浏览器扩展中             |
|                                                  |
| [保存设置] [清除配置]                             |
+--------------------------------------------------+
```

### 7.3 核心字段

- `文本翻译配置`：始终存在，是文本选区翻译与粘贴翻译的基础能力。
- `文本翻译供应商类型`：预置供应商或自定义兼容地址。
- `文本翻译服务地址`：对文本翻译可编辑。
- `文本翻译 API Key`：掩码输入，可 reveal。
- `文本模型`：必填，仅服务文本选区翻译和粘贴翻译。
- `截图翻译模式`：
  - `直接多模态翻译`
  - `提取后翻译`
- 当模式为 `直接多模态翻译` 时：
  - 可选择 `复用文本翻译配置` 或 `单独配置截图直译模型`。
- 当模式为 `提取后翻译` 时：
  - `截图提取配置`：必填，用于截图文字提取。
  - `截图翻译配置`：可选择 `复用文本翻译配置` 或 `单独配置截图翻译模型`。
- `默认目标语言`：常用语言下拉。
- `权限状态`：按实际启用阶段分别显示。
- `验证状态`：按实际启用阶段分别显示。

### 7.4 设置页状态

- 初始加载态：读取本地配置中。
- 文本未配置态：文本翻译不可用。
- 截图未配置态：截图翻译不可用。
- 直译模式就绪态：文本翻译已配置，截图直译配置完整。
- 两阶段模式部分就绪态：截图提取已配置但截图翻译未配置，或反之。
- 已配置未验证态：允许用户保存，但提示对实际启用阶段分别验证。
- 分项验证成功态：显示 `文本翻译可用`、`截图直译可用`、`截图提取可用`、`截图翻译可用` 中的对应项。
- 单侧权限缺失态：优先提示授权对应阶段的服务地址，而不是直接尝试请求。

## 9. 视觉风格建议

- 风格方向：轻量、克制、工具型。
- 保持中性色为主，主色用于主操作。
- API Key 输入区域应有更强的安全感表达，例如掩码、说明文案、局部提示。
- 截图遮罩使用半透明深灰或黑色，保证框选区域足够醒目。

## 10. 交互细节

### 10.1 复制反馈

- 点击 `复制译文` 后，按钮短暂切换为 `已复制`。

### 10.2 键盘交互

- popup 输入框支持常规粘贴快捷键。
- 可支持 `Cmd/Ctrl + Enter` 触发文本翻译。
- 截图模式下 `Esc` 必须退出。

### 10.3 设置页敏感交互

- API Key 默认掩码。
- reveal 动作是短期显示，不建议默认长期展开。
- `清除配置` 需二次确认。

## 11. 可访问性要求

- 所有按钮都需要明确文本或 aria-label。
- Popup tab 切换不使用可点击 `div`，而使用可访问按钮语义。
- 焦点态需要可见。
- 错误信息需配合图标或文字提示。
- 原文和译文都应允许文本选择与复制。
- 长 URL、长模型名、长英文都不应挤压布局。

## 12. UI 状态矩阵

### 12.1 页面文本浮层

| 状态 | 是否显示按钮 | 是否显示浮层 | 主要操作 |
| --- | --- | --- | --- |
| 无选区 | 否 | 否 | 无 |
| 有选区未触发 | 是 | 否 | 点击翻译 |
| 未配置 | 是 | 是 | 打开设置 |
| 翻译中 | 否 | 是 | 关闭 |
| 翻译成功 | 否 | 是 | 复制译文、重试、关闭 |
| 翻译失败 | 否 | 是 | 重试、关闭 |

### 12.2 页面截图模式

| 状态 | 是否显示遮罩 | 是否允许框选 | 主要操作 |
| --- | --- | --- | --- |
| Idle | 否 | 否 | 从 popup 启动 |
| 未配置 | 否 | 否 | 打开设置 |
| Armed | 是 | 是 | 拖拽框选、Esc 取消 |
| Selecting | 是 | 是 | 继续拖拽 |
| Processing | 是 | 否 | 等待、关闭 |
| Success | 否 | 否 | 复制译文、重试、关闭 |
| Error | 否 | 否 | 重试、关闭 |

### 12.3 Popup

| 状态 | 当前 tab | 主操作 |
| --- | --- | --- |
| 初始态 | 文本翻译 | 输入文本或切换截图 |

## 13. 文案风格规范

- 使用简洁中文。
- 优先动作导向文案，例如 `翻译`、`复制译文`、`保存设置`、`开始截图翻译`。
- 配置问题优先明确告诉用户下一步，例如 `请先完成文本翻译配置`、`请先完成截图直译配置`、`请先完成截图提取配置`、`请先完成截图翻译配置`、`请先授权对应服务地址`。
- 避免技术细节文案，例如 `Fetch failed`、`500 Internal Server Error`。

## 14. 与架构文档的关系

- `docs/architecture.md` 定义系统职责、目录和内部接口边界。
- 本文档定义文本选区浮层、截图模式、popup 和设置页的界面结构、组件关系和状态表现。
- 后续实现时，目录与模块边界应以架构文档为准，UI 行为与视觉层级应以本文档为准。
